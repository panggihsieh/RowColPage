# 八格簿

這是「八格簿」第二版草稿的 Web App，延續第一版的 A4 八格簿排版，並補上更適合班級實際使用的欄位與設定保存功能。

目前版本提供 A4 直式排版、每頁 8 格、4 列 2 欄的版面配置，格子編號依照左到右、上到下自動排序，並可直接列印或另存 PDF。

## 版本資訊

- 版本名稱：八格簿
- 版本定位：第二版草稿
- 版面格式：A4 直式
- 格數配置：每頁 8 格
- 排列方式：4 列 2 欄
- 編號順序：左到右、上到下
- 開發分支：`codex/v2-draft`

## 可否在本地端執行

可以。

這個專案是純前端靜態網頁，沒有後端服務相依，因此可直接在本地端執行。

本地端執行方式有兩種：

### 方式一：直接開啟檔案

直接用瀏覽器開啟 `index.html` 即可。

### 方式二：使用 localhost

在專案目錄執行：

```powershell
python -m http.server 8000
```

再用瀏覽器開啟：

```text
http://localhost:8000
```

如果電腦已安裝 Python，這是最簡單穩定的本機執行方式。

## 如何避免列印時多出一頁

這個專案先前曾經發生「明明只做一張 A4，列印預覽卻多出一頁」的問題。

### 之前發生的原因

當時頁面中的格子高度是用固定公式直接計算，公式預設了頁首的高度大致不變。後來新增了右上角的「家長簽名」文字與長方形框線，頁首實際高度變高，但格子仍沿用原本的固定高度，結果整張內容總高度超過 A4 頁面。

瀏覽器在列印時只要發現內容超出 `297mm`，就會自動把多出的部分切到下一頁，因此看起來就像「多了一頁」。

### 現在的處理方式

目前程式已改成以下做法：

- `.page` 固定為 A4 高度
- `.grid` 使用 `flex: 1` 佔滿剩餘空間
- `.grid` 使用 `grid-template-rows: repeat(4, minmax(0, 1fr))`
- `.cell` 不再寫死固定高度，而是平均分配可用空間

這樣即使頁首或頁尾區塊有些微增減，格子區也會自動縮放，不容易再把內容撐到下一頁。

### 後續編寫時的注意事項

- 不要把格子高度寫死成固定毫米或固定公式，除非連頁首、頁尾高度也一起重新計算
- 新增頁首元素時，要注意是否會擠壓到格子區的總高度
- 新增頁尾元素時，要注意是否會推高整張頁面
- 列印版型請優先用「剩餘空間分配」而不是「每塊高度硬算」
- 每次調整 A4 版面後，都要實際打開列印預覽確認是否溢頁
- 若新增區塊是可顯示/隱藏的功能，兩種狀態都要檢查一次列印結果
