# RowColPage

RowColPage 是一組可列印的 A4 練習簿 Web App，目前包含四個版本：

- 第一版：八格簿
- 第二版：多格簿 / 自學筆記
- 第三版：幾何練習簿
- 第四版：克隆原始數學題目

整個專案採用純前端靜態網頁實作，可直接在瀏覽器開啟，也可透過本機伺服器或 GitHub Pages 使用，再列印或另存 PDF。

## 版本一覽

### 第一版：八格簿

特色：

- A4 直式
- 固定八格
- 編號由左到右、由上到下
- 支援多種輔助線
- 適合作為基礎數學作業簿

### 第二版：多格簿 / 自學筆記

特色：

- A4 直式
- 可自訂欄數與列數
- 每頁格數不再固定為八格
- 第一頁顯示頁首資訊，後續頁只保留格子與頁碼
- 支援家長簽名顯示切換

### 第三版：幾何練習簿

第三版已完成目前規劃的核心流程，主打「逐題整理幾何題目 + 右側保留手寫計算區」。

特色：

- A4 直式
- 單欄逐題排列，每頁依列數決定題數
- 每題左側為題目區，右側為計算程序區
- 支援標題、姓名、日期、起始編號、頁數、列數設定
- 支援家長簽名顯示切換
- 支援每題獨立上傳圖片
- 支援已上傳圖片的更換與清除
- 支援先複製截圖，再點選目標格後按 `Ctrl+V` 貼上
- 支援按下「貼上題圖」按鈕，直接從剪貼簿帶入圖片
- 題圖會自動縮放到題目區內
- 列印時自動隱藏題號與上傳控制列，只保留排版內容
- 每格輔助線支援：橫線、點、無
- 預設輔助線為：無

### 第四版：克隆原始數學題目

第四版主打「上傳含有題號的圖片或 PDF，根據題號規則自動分題，再透過勾選頁挑題後貼到左側題目區，克隆原始數學題目做成複習單」。

特色：

- A4 直式
- 單欄逐題排列，每頁依列數決定題數
- 左側為題目區，右側為計算程序區
- 支援上傳含有題號的數學圖片或 PDF
- 支援以每行一條 RegExp 或關鍵字定義題號規則
- 支援設定左右界、標題上方留白、頁尾保留、下一題前間隔
- 支援依標題自動裁出題目區塊
- 支援開啟新勾選頁依題號挑選題目
- 支援將勾選到的題目依原順序帶入左側題目區
- 若題數超過目前頁數，會自動補足頁面
- 保留逐題手動上傳、貼上、更換、清除圖片能力
- 專案內附 `v4/範例.pdf`，第四版首頁會預設載入此檔案，可直接做抽題測試
- 適合把整份數學講義、考卷或題本快速整理成可書寫的原題複習單

## 第三版使用重點

第三版目前建議流程如下：

1. 設定標題、姓名、日期、起始編號、頁數、列數。
2. 點選欲放入題圖的目標格。
3. 使用以下任一方式帶入圖片：
   - `上傳題圖`
   - `更換題圖`
   - `貼上題圖`
   - 直接按 `Ctrl+V`
4. 確認左側題目區圖片與右側計算程序區排版是否正確。
5. 若需要調整，重新更換圖片或清除該題。
6. 點選 `列印 / 另存 PDF` 輸出。

## 為什麼列印時可能多一頁

專案早期曾遇到「只加了一點頁首內容，就多出一頁」的情況。原因通常是：

- 頁面總高度仍以舊公式計算
- 但頁首或控制內容已增加
- 導致整體內容超過 A4 高度
- 瀏覽器列印時就會自動分到下一頁

目前的做法是：

- 讓 `.page` 固定為 A4 高度
- 讓 `.grid` 使用剩餘空間自動分配
- 避免每格高度用硬編碼公式計算

之後若再新增頁首欄位或題目區，仍要注意整頁總高度不能超出 A4。

## 未來設計建議

第四版屬於「從 PDF / 圖片自動抽題，再產生可列印練習單」的半自動工具。這類工具容易受到來源檔案格式、題號規則、OCR、裁切邏輯與瀏覽器效能影響，因此未來若繼續擴充第四版，或設計類似軟體，建議採用「可預覽、可修正、可回復」的流程，而不是追求一次全自動完成。

### 常見錯誤來源

- 題號格式不一致，導致系統無法穩定判斷每一題起點。
- 題號規則太寬，可能把頁碼、選項、段落編號或頁尾文字誤判為題目。
- 題號規則太窄，遇到不同題號格式時會漏抓題目。
- PDF 可能沒有文字層，掃描型 PDF 看得到文字，但程式無法直接讀取。
- 多欄、表格、浮水印、頁首頁尾與裝飾元素會干擾題目切割。
- OCR 對數學符號、分式、上下標與低解析度圖片不一定穩定。
- 自動裁切依賴題號位置、下一題位置、左右界與頁尾設定，版面不規則時容易裁錯。
- 大型 PDF、多頁圖片或大量 Data URL 會造成瀏覽器記憶體與效能壓力。
- 主頁與勾選頁若分開，可能受到彈出視窗、跨頁通訊或暫存資料影響。
- 錯誤訊息若只顯示「分析失敗」，使用者無法判斷該修改檔案、規則、裁切參數或 OCR 設定。

### 建議規劃流程

未來流程建議拆成明確階段：

```text
上傳來源
  ↓
來源檢測
  ↓
題號規則設定與預覽
  ↓
版面範圍設定
  ↓
自動分析與分題
  ↓
逐題裁切確認
  ↓
OCR / AI 文字辨識
  ↓
人工修正文字
  ↓
題目勾選
  ↓
套用到練習單
  ↓
列印 / 匯出 PDF
```

每一步都應顯示目前狀態、發現的問題、建議修正方式、下一步按鈕與返回修改入口。分析完成不應等於可列印完成，仍應經過裁切確認、文字修正與題目勾選。

### 建議系統架構

類似工具可拆成以下模組：

```text
使用者介面層 UI
  ↓
流程控制層 Workflow Controller
  ↓
來源處理層 Source Processor
  ↓
題號辨識層 Question Detector
  ↓
裁切處理層 Crop Engine
  ↓
OCR / AI 辨識層 Recognition Engine
  ↓
題庫資料層 Question Store
  ↓
練習單排版層 Worksheet Renderer
  ↓
列印匯出層 Print / Export
```

各層建議職責如下：

- `Source Processor`：載入 PDF / 圖片、判斷是否有文字層、產生頁面預覽與文字行資料。
- `Question Detector`：根據題號規則找出題目起點，並提供匹配預覽與排除規則。
- `Crop Engine`：根據題號位置、下一題位置、左右界、頁首與頁尾設定產生裁切框，並保存人工修正結果。
- `Recognition Engine`：執行 OCR 或讀取 PDF 文字層，產生可人工修正的題目文字。
- `Question Store`：保存原始頁面、裁切框、題圖、辨識文字、人工修正文字、確認狀態與勾選狀態。
- `Worksheet Renderer`：將已勾選題目依序放入 A4 練習單，並自動補足頁面。
- `Print / Export`：檢查列印版面、隱藏操作 UI，輸出穩定的 A4 直式結果。

### 避免類似錯誤的設計原則

- 題號規則要能即時預覽，讓使用者在正式分析前先確認抓到哪些行。
- 來源檔案要先檢測，區分文字型 PDF、掃描型 PDF 與一般圖片。
- 複雜版面要提供單欄、雙欄、自訂區域或每頁獨立區域模式。
- 自動裁切只能視為初稿，每題都要能人工調整與還原。
- OCR 文字只能作為輔助，原始題圖必須永遠保留。
- 每題應保存原始頁面、題號起點、裁切框、裁切圖、辨識文字與確認狀態。
- 大檔案應支援分頁、分批分析、進度顯示與取消操作。
- 主頁與勾選頁若分離，必須顯示同步狀態，並清楚區分目前資料與歷史暫存資料。
- 錯誤提示要指出原因與下一步，例如「找不到題號，請修改題號規則」比「分析失敗」更有效。
- 編輯版面與列印版面應分離，列印前要檢查是否超出 A4 或產生空白頁。

### 核心結論

這類軟體最重要的設計方向不是完全自動化，而是半自動化：

```text
自動分析
  ↓
即時預覽
  ↓
逐步確認
  ↓
人工修正
  ↓
穩定輸出
```

只要每一個自動結果都能被看見、被修改、被回復，即使來源 PDF 格式混亂、OCR 辨識不準或題號規則有誤，也能在流程中修正，而不是最後才產生錯誤的練習單。
